## Overview

This directory contains a complete Verilog implementation of the **SPI (Serial Peripheral Interface)** protocol, designed for FPGA deployment on the Nexys A7 board. SPI is a synchronous, full-duplex serial communication protocol widely used for communication between microcontrollers and peripheral devices.

The implementation includes:
- **SPI Master Controller** - Initiates and controls communication
- **SPI Slave Controller** - Responds to master commands
- **Top-level Integration Module** - Hardware interface for Nexys A7 board

## SPI Protocol Basics

### What is SPI?

SPI is a **four-wire** serial protocol that enables **full-duplex** communication between a master and one or more slave devices. It is commonly used for:
- SD cards and flash memory
- Real-time clocks (RTC)
- Temperature sensors
- Display controllers
- ADC/DAC modules

### Key Characteristics

| Feature | Description |
|---------|-------------|
| **Communication Mode** | Full-duplex (simultaneous send and receive) |
| **Topology** | Master-Slave (one master, one or more slaves) |
| **Clock Control** | Master generates the clock (SCLK) |
| **Data Rate** | Programmable, typically 1-100 Mbps |
| **Distance** | Short-distance (board-level communication) |

### SPI Signals

The SPI interface consists of four main signals:

1. **SCLK (Serial Clock)** - Generated by the master, synchronizes data transfer
2. **MOSI (Master Output, Slave Input)** - Data line from master to slave
3. **MISO (Master Input, Slave Output)** - Data line from slave to master
4. **CS_N (Chip Select)** - Active-low signal that selects the slave device

### SPI Modes

SPI has four modes based on clock polarity (CPOL) and clock phase (CPHA):

| Mode | CPOL | CPHA | Description |
|------|------|------|-------------|
| **Mode 0** | 0 | 0 | Clock idles low; data captured on rising edge (This Implementation) |
| Mode 1 | 0 | 1 | Clock idles low; data captured on falling edge |
| Mode 2 | 1 | 0 | Clock idles high; data captured on falling edge |
| Mode 3 | 1 | 1 | Clock idles high; data captured on rising edge |

**This implementation uses SPI Mode 0 (CPOL=0, CPHA=0).**

## Implementation Details

### Clock Generation (SPI Master)

The SPI master generates the SCK clock using a **programmable clock divider**:

```verilog
parameter integer CLOCK_DIV = 50;
```

**Formula:**
```
SCK_Period = 2 × CLOCK_DIV × System_Clock_Period
SCK_Frequency = System_Clock_Frequency / (2 × CLOCK_DIV)
```

**Example:**
- System Clock: 100 MHz (10 ns period)
- CLOCK_DIV: 50
- SCK Frequency: 100 MHz / (2 × 50) = **1 MHz**
- SCK Period: 1000 ns

### Data Transfer Protocol

**SPI Mode 0 (CPOL=0, CPHA=0):**

1. **CS_N transitions LOW** - Slave is selected
2. **Data is sampled on SCLK rising edge** - Receiver captures data
3. **Data is shifted on SCLK falling edge** - Sender presents next bit
4. **8 bits transferred** - Full byte exchange
5. **CS_N transitions HIGH** - Transfer complete, slave deselected

### State Machine

Both master and slave implementations use **Finite State Machines (FSM)** to control the protocol:

#### Master FSM
```
┌──────────┐
│  IDLE    │◄─────────────────┐
└────┬─────┘                  │
     │ start=1                │
     ▼                        │
┌──────────┐                  │
│ TRANSFER │ ─────────────────┤
└────┬─────┘  bit_cnt=8       │
     │                        │
     ▼                        │
┌──────────┐                  │
│  DONE    │──────────────────┘
└──────────┘
```

#### Slave FSM

The slave does not use explicit states but operates reactively:
- Waits for CS_N to go LOW
- Detects SCLK edges
- Transfers data on each clock cycle
- Signals completion when CS_N goes HIGH

## Module Descriptions

### 1. **spi_master.v** - SPI Master Controller

**Purpose:** Initiates SPI transfers and controls the communication protocol.

**Parameters:**
```verilog
parameter integer CLOCK_DIV = 50  // Clock divider for SCK generation
```

**Inputs:**
- `clk` - System clock (100 MHz)
- `rst_n` - Active-low reset
- `start` - Pulse to initiate transfer
- `tx_data[7:0]` - Data to transmit
- `miso` - Slave data input

**Outputs:**
- `mosi` - Master data output
- `sclk` - Serial clock
- `cs_n` - Chip select (active-low)
- `rx_data[7:0]` - Received data
- `busy` - Transfer in progress flag
- `done` - Transfer complete flag

**Key Features:**
- Programmable clock divider for flexible baud rate
- Full-duplex simultaneous transmission and reception
- 8-bit data words
- Active-low chip select
- Status signals for busy and done states

**Functional Description:**

1. **Clock Division:** Generates SCLK by toggling div_tick on counter overflow
2. **Edge Detection:** Detects SCLK rising and falling edges for synchronization
3. **Data Shifting:** 
   - MOSI updated on SCLK falling edge
   - MISO sampled on SCLK rising edge
4. **Bit Counting:** Tracks number of bits transferred (0-7)
5. **State Management:**
   - ST_IDLE: Waiting for start signal
   - ST_TRANSFER: Actively transferring bits
   - ST_DONE: Transfer complete, preparing for next cycle

---

### 2. **spi_slave.v** - SPI Slave Controller

**Purpose:** Receives commands from master and responds accordingly.

**Inputs:**
- `clk` - System clock
- `rst_n` - Active-low reset
- `sclk` - Serial clock from master
- `cs_n` - Chip select from master
- `mosi` - Master data input
- `tx_data[7:0]` - Data to transmit

**Outputs:**
- `miso` - Slave data output
- `rx_data[7:0]` - Received data
- `data_valid` - One-cycle pulse when byte received

**Key Features:**
- Responds to master-provided clock
- Detects chip select edges
- Automatic data buffering
- Data valid signal for synchronization
- Edge detection for robust operation

**Functional Description:**

1. **Edge Detection:**
   - Detects SCLK rising and falling edges
   - Detects CS_N falling edge for transaction start

2. **Data Reception:**
   - Shifts in MOSI bits on SCLK rising edges
   - Accumulates bits in rx_shift register
   - Captures complete byte when bit_cnt reaches 7

3. **Data Transmission:**
   - Preloads MSB when CS_N goes LOW
   - Shifts out remaining bits on SCLK falling edges

4. **State Tracking:**
   - When CS_N is HIGH: Reset counters and preload tx_data
   - When CS_N is LOW: Process data according to SCLK edges

---

### 3. **spi_top.v** - Top-Level Integration Module

**Purpose:** Interfaces SPI master with Nexys A7 board I/O.

**Generics:**
```verilog
parameter integer CLOCK_DIV = 50
```

**Board I/O Connections:**

| Signal | Pin Type | Function |
|--------|----------|----------|
| CLK100MHZ | Input | 100 MHz board clock |
| CPU_RESETN | Input | Active-low reset button |
| BTNC | Input | Push button to start SPI |
| SW[7:0] | Input | Data to transmit (switches) |
| spi_mosi | Output | Master data out |
| spi_sclk | Output | Serial clock |
| spi_cs_n | Output | Chip select |
| spi_miso | Input | Slave data in |
| LED[15:0] | Output | Status display |

**LED Mapping:**
- LED[7:0] - Received SPI data
- LED[8] - Busy flag
- LED[9] - Done flag
- LED[15:10] - (Reserved)

**Functional Description:**

1. **Reset Handling:** Passes CPU_RESETN to SPI master
2. **Button Detection:** Detects BTNC falling edge and generates start pulse
3. **Data Input:** Reads SW[7:0] as transmit data
4. **Master Instantiation:** Connects SPI master with appropriate clock divider
5. **Status Display:** 
   - Shows received data on LED[7:0]
   - Displays busy status on LED[8]
   - Displays done status on LED[9]

## Usage

### 1. Simulation (Using Xilinx Vivado/ISim)

```bash
# Launch Vivado and open the project
# Run simulation with tb_spi.v
# Observe waveforms for:
#   - SCLK generation and proper frequency
#   - CS_N timing relative to data transfer
#   - MOSI/MISO data bits and bit ordering
#   - State transitions in master FSM
```

### 2. FPGA Deployment (Nexys A7)

**Hardware Setup:**
1. Connect Nexys A7 board to computer via USB
2. Connect external SPI slave device:
   - MOSI (Pin) → Slave DIN
   - MISO (Pin) → Slave DOUT
   - SCLK (Pin) → Slave CLK
   - CS_N (Pin) → Slave CS

**Build and Program:**
```tcl
# In Vivado:
1. Create project with Nexys A7 board
2. Add RTL files (spi_master.v, spi_slave.v, spi_top.v)
3. Set spi_top.v as top module
4. Run synthesis and place & route
5. Generate bitstream
6. Program FPGA
```

**Operation:**
- **Press BTNC** (center button) to start SPI transfer
- **Set SW[7:0]** to choose data to send
- **Observe LED[7:0]** to see received data
- **LED[8]** indicates transfer in progress (busy)
- **LED[9]** indicates transfer complete (done)

### 3. Parameter Configuration

To change SPI clock frequency:

```verilog
// In spi_top.v instantiation
spi_master #(
    .CLOCK_DIV(100)  // Change this value
) u_spi_master (
    ...
);
```

**Clock Calculation:**
```
Desired_SCLK = 1 MHz
System_Clock = 100 MHz

CLOCK_DIV = System_Clock / (2 × Desired_SCLK)
CLOCK_DIV = 100 MHz / (2 × 1 MHz) = 50
```

## Testing

### Simulation Test Cases

The testbench (`tb_spi.v`) validates:

1. **Clock Generation**
   - Verify SCLK frequency = System_Clock / (2 × CLOCK_DIV)
   - Verify SCLK duty cycle ≈ 50%

2. **Data Transfer**
   - Transmit known data patterns (0xA5 = 10100101, 0x5A = 01010101)
   - Verify MOSI presents bits MSB-first
   - Verify MISO samples correctly
   - Check rx_data matches transmitted pattern

3. **Timing Compliance**
   - MOSI stable during SCLK high periods
   - Data changes only on SCLK falling edges
   - Sampling occurs on SCLK rising edges

4. **Control Signals**
   - CS_N goes LOW at transfer start
   - CS_N goes HIGH at transfer end
   - Busy flag set during transfer
   - Done flag pulses for one cycle at completion

5. **State Machine**
   - Transitions from IDLE → TRANSFER → DONE → IDLE
   - Remains in states for correct durations
   - Bit counter increments properly
